Onelist MVP Launch Plan

Document Version: 2026-01-30 Status: Active Development Focus: Rapid MVP launch with OpenClaw integration, Beacon CMS, and community forum

1. Executive Summary

1.1 Vision

Launch Onelist as the "augmented memory" platform with three deployment models: 1. Self-hosted (Free) - OpenClaw users and privacy-focused developers 2. Cloud Sync ($3/mo + usage) - E2EE cloud backup with public pages 3. Web Access ($20/mo) - Fully managed with Beacon sites and agents

3. Architecture Overview

3.0 Agent Implementation Standards

REQUIRED: Before implementing any agent, consult the Agent Implementation Reference guides in Appendix B.
Core Principles (all agents):
  • Passive Context Strategy: Embed knowledge indexes in prompts over active retrieval
  • SQ-BCP Precondition Checking: Track Sat/Viol/Unk states for uncertain actions
  • Detailed Tool Definitions: 3-4+ sentence descriptions with input examples
  • Observability from Day One: Trace multi-step workflows, log tool calls
  • Use Unified Modules: All agents MUST use the Unified Agent Modules for telemetry, security, model selection, and cost tracking
    • Agent Run Styles:

    Not all agents need to be always-running. Each agent's run style should be evaluated during planning:

    AgentRun Style(s)TriggerNotes
    RiverAlways-running, LoopingUser conversationOnly agent requiring persistent loop; user-facing
    ReaderEvent-triggeredEntry create/updateOban worker; processes then exits
    SearcherOn-demand + Event-triggeredQuery request / Entry createQuery-time search; background embedding
    Asset EnrichmentEvent-triggeredAsset uploadOban worker; processes then exits
    FeederEvent-triggered + ScheduledClip request / RSS scheduleOn-demand clips; scheduled feed polling
    Run Style Definitions:
  • Always-running: Persistent process with conversation loop (River only)
  • Event-triggered: Spawned by Oban on specific events, processes, then exits
  • On-demand: Invoked synchronously at request time
  • Scheduled: Runs on cron-like schedule (Oban cron)
  • Orchestrated: Invoked by River as a sub-agent/tool

    • River may orchestrate other agents as tools for complex multi-step tasks, but those agents don't maintain their own loops.

    CREATE TABLE entry_links (
  id UUID PRIMARY KEY,
  source_entry_id UUID NOT NULL REFERENCES entries(id) ON DELETE CASCADE,
  target_entry_id UUID NOT NULL REFERENCES entries(id) ON DELETE CASCADE,
  link_type VARCHAR(50) NOT NULL,
  metadata JSONB,
  inserted_at TIMESTAMPTZ NOT NULL,
  updated_at TIMESTAMPTZ NOT NULL
);

    CREATE INDEX entry_links_source_idx ON entry_links(source_entry_id);
CREATE INDEX entry_links_target_idx ON entry_links(target_entry_id);
CREATE INDEX entry_links_type_idx ON entry_links(link_type);

Appendix C: Detailed R2 Cost Calculator

Monthly R2 Cost = (Storage GB × $0.015) + (Class A ops / 1M × $0.36) + (Class B ops / 1M × $0.036)

Example: 1000 users, 500 MB avg storage, 100 uploads/mo, 500 downloads/mo


Storage:  1000 × 0.5 GB × $0.015 = $7.50
Class A:  1000 × 100 / 1,000,000 × $0.36 = $0.036
Class B:  1000 × 500 / 1,000,000 × $0.036 = $0.018
──────────────────────────────────────────────────
Total: $7.55/month for 1000 users
Per user: $0.00755/month (less than 1 cent)
