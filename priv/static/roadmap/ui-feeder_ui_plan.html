Feeder Agent UI Plan

Document Version: 2026-01-30 Status: Active Parent: ui_plan.md Related: shared_agent_ui_components.md, settings_ui_plan.md

1. Overview

This document defines UI components specific to the Feeder Agent. For shared agent UI patterns (processing status, activity items, list view, duplicate detection), see shared_agent_ui_components.md.

Feeder Responsibilities:
  • Web Clipper (MVP)
  • RSS feed subscriptions
  • External service sync (Evernote, Notion, Obsidian)
  • One-time file imports
  • Format conversion
    • Key Design Decision: Sources as Entries

    All Feeder sources are stored as entries, not as separate configuration objects:

  • Entry type: feeder-source (RSS feeds, service connections)
  • Entry type: feeder-import-job (one-time imports)
  • Source type stored in metadata: rss, evernote, notion, obsidian, etc.

    • This allows sources to be organized using existing Library features (tags, search, list view).

    2. Entry Types

    2.1 feeder-source

    Represents a continuous sync source (RSS feed, service connection).

    {
  "entry_type": "feeder-source",
  "title": "Hacker News - Best",
  "metadata": {
    "source_type": "rss",
    "feed_url": "https://hnrss.org/best",
    "sync_frequency_minutes": 60,
    "sync_enabled": true,
    "last_sync_at": "2026-01-30T10:00:00Z",
    "last_sync_status": "success",
    "items_imported": 45,
    "credentials": { ... }  // encrypted
  }
}

    2.2 feeder-import-job

    Represents a one-time import operation.

    {
  "entry_type": "feeder-import-job",
  "title": "work_notes.enex",
  "metadata": {
    "source_type": "evernote_enex",
    "file_size_bytes": 52428800,
    "status": "completed",
    "progress_percent": 100,
    "items_total": 1247,
    "items_succeeded": 1245,
    "items_failed": 2,
    "errors": [ ... ]
  }
}
