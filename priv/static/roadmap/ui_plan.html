1.3 Design Goals

1. Every piece of data is visible - Users can see entries, memories, tags, enrichments, processing status 2. AI is transparent - Users see what AI extracted, can accept/reject suggestions 3. Clean default experience - Power features tucked away until needed 4. Fast and responsive - Instant feedback, optimistic updates, smooth animations 5. Works offline - Desktop app functions without internet (sync when available)

    6.2 Memory Visibility Rules

    All memories extracted by AI are visible to users:

    1. Per-Entry View: Memories tab shows all memories from that entry 2. Global View: Memories page shows all memories across all entries 3. Superseded Memories: Shown with strikethrough and explanation 4. Confidence Scores: Always visible 5. Source Link: Every memory links back to source entry 6. Entities: Extracted entities shown as chips 7. Temporal Resolution: Show original expression + resolved date

    Read-Only Rationale:
  • Memories are derived data, not primary data
  • Editing memories would create inconsistency with source
  • To "fix" a memory, user edits source entry â†’ triggers re-processing

    • 6.3 Processing Transparency

    Users always know what's being processed:

    1. Queue Visibility: "3 items processing" indicator 2. Progress Display: Real-time progress for long operations 3. Cost Display: Per-operation and cumulative costs shown 4. Error Visibility: Failed operations shown with retry option 5. Model Attribution: Which AI model processed each item
