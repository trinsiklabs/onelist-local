Onelist Corporate Website Plan

Overview

The Onelist corporate website at onelist.my serves as the central hub for marketing, documentation, user accounts, and billing. Built with Beacon CMS for Phoenix, it combines content management flexibility with the performance and real-time capabilities of Phoenix LiveView.

Domain Structure

DomainPurpose
onelist.myCorporate website, marketing, docs
app.onelist.myAccount dashboard, billing, device management
[username].onelist.myVM tier user instances (full web UI)
api.onelist.myHeadless cloud API (sync, storage)

Technology Stack

Beacon CMS

  • Phoenix-native CMS with LiveView
  • Visual page builder for marketing pages
  • Content management for docs/blog
  • SEO-friendly with static generation options
  • Custom components for interactive elements

    • Why Beacon?

  • Same tech stack as Onelist Core (Elixir/Phoenix)
  • No separate CMS infrastructure needed
  • LiveView for interactive pricing calculators, demos
  • Easy to extend with custom components
  • Open source, self-hostable

    • Feature: E2EE Sync (onelist.my/features/sync)

    Purpose: Explain zero-knowledge sync and build trust around privacy. Sections:

    1. Hero

  • Headline: "Sync Everywhere. Trust No One."
  • Subheadline: "End-to-end encrypted sync means your data is protected even from us."

    • 2. What E2EE Means

  • Data encrypted on your device before upload
  • We store encrypted blobs we cannot read
  • Only your passphrase can decrypt
  • Even if we're hacked, your data is safe

    • 3. Trust Model Comparison | Service | Trust Required | |---------|----------------| | Google Drive | Trust Google completely | | Dropbox | Trust Dropbox completely | | iCloud | Trust Apple completely | | Onelist | Trust no one (E2EE) |

    4. How It Works (diagram) 

       Your Device                Onelist Cloud              Your Other Device
   ───────────                ────────────               ─────────────────
   1. Write entry
   2. Encrypt with your key
   3. Upload blob ─────────► 4. Store blob ────────────► 5. Download blob
                              (cannot read)              6. Decrypt with key
                                                        7. Read entry

    5. Sync Options

  • Free tier: P2P sync (device-to-device)
  • Cloud tier: Always-available encrypted sync
  • Self-hosted: Your own infrastructure

    • 6. Security Details

  • Encryption: AES-256-GCM
  • Key derivation: Argon2id from passphrase
  • Perfect forward secrecy
  • Open source, auditable

    • 7. CTA

  • "Read our security whitepaper" → /security
  • "Get Started" → Signup

    • Feature: Hybrid Search (onelist.my/features/search)

    Purpose: Explain the power of combined full-text + semantic search. Sections:

    1. Hero

  • Headline: "Find Anything. Instantly."
  • Subheadline: "Full-text search meets AI-powered semantic understanding."

    • 2. The Problem with Traditional Search

  • Keyword search misses context
  • "What was that article about productivity?" fails
  • Can't search by meaning, only by words

    • 3. Hybrid Search Explained | Search Type | What It Finds | Example | |-------------|---------------|---------| | Full-text | Exact words | "PostgreSQL tutorial" | | Semantic | Similar meaning | "database learning resources" | | Hybrid | Best of both | Finds both matches, ranks intelligently |

    4. How It Works

  • PostgreSQL FTS for keyword matching
  • pgvector for embedding similarity
  • Combined scoring and ranking
  • All happens locally (privacy preserved)

    • 5. Search Examples (interactive demo?)

  • Query: "that meeting about the product launch"
  • Results: Meeting notes, related emails, action items

    • 6. Powered By

  • PostgreSQL + pgvector
  • Local embedding generation (configurable model)
  • Runs entirely on your device or server
