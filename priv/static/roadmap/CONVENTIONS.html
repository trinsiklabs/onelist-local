Roadmap Conventions

Plan File Status Tracking

Frontmatter (always)

Every plan file should have status in the YAML frontmatter: 
---
status: active | completed | on-hold
completed_at: 2026-01-30  # when status is completed

Archive (periodically)

When a batch of plans is done, move completed files to: 
roadmap/completed/

This keeps the active roadmap clean while preserving history.

Status Values

  • active — currently being worked on or next up
  • completed — done, ready for archive
  • on-hold — paused, not abandoned

Index Generation

The roadmap index generator should: 1. List active plans from roadmap/.md 2. List completed plans from roadmap/completed/.md (collapsed or separate section)
