Static Site Desktop Page Updates

Document: static_site/features/desktop.html Created: 2026-01-29 Purpose: Align marketing with technical decisions

Summary of Required Changes

The static site currently describes a "pure Swift/SwiftUI native app" with performance claims that don't match the planned Tauri + Phoenix architecture. These changes bring marketing into alignment with technical reality while maintaining competitive positioning.

1. Technology Description Changes

Current (Remove)

"Written entirely in Swift using Apple's modern SwiftUI framework"
"No Electron. No JavaScript runtime. No Node.js. Just pure, compiled Swift code"

Replace With

"A native macOS app with deep system integration"
"Built for macOS with Spotlight search, menu bar access, and global shortcuts"
"3x lighter than Electron apps - no embedded Chromium browser"
Rationale: Honest about architecture while maintaining competitive positioning against Electron.

2. Performance Claims Updates

Current → New

MetricCurrent ClaimNew Claim
Launch time~0.5 seconds3-5 seconds (full offline database)
Memory usage<80 MB~250 MB (includes local PostgreSQL)
App size~25 MB~85 MB (3x smaller than Electron)
CPU idle0%Minimal background sync

Suggested Copy

Performance built for real work:
• ~85 MB app size (3x smaller than Electron apps)
• ~250 MB memory with full local database
• Instant offline access - your data never waits for the network
• Background sync that won't drain your battery
Framing: Emphasize the VALUE of having a full local database rather than apologizing for resource usage.
