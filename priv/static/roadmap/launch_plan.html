1. Executive Summary

1.1 Goal

Launch Onelist MVP as quickly as possible while ensuring:

  • Sustainable unit economics (don't lose money per user)
  • Pricing that doesn't need to change dramatically post-launch
  • Infrastructure that scales without surprises

    2. Launch Strategy: Phased Approach

    Phase 0: Soft Launch (Week 1-2)

  • Deploy to production infrastructure
  • Invite-only beta (friends, family, early supporters)
  • Validate core flows work
  • Monitor costs closely

    • Phase 1: Limited Launch (Week 3-4)

  • Open signups with waitlist
  • Control growth to monitor unit economics
  • Adjust pricing if needed before wide launch

    • Phase 2: Public Launch (Week 5+)

  • Remove waitlist
  • Marketing push
  • Monitor and optimize
    • Why phased? Catch pricing/cost issues before they scale.

    Web Access cost breakdown:
  • Fly.io shared-cpu-1x, 1GB: ~$6/mo
  • PostgreSQL (shared/free tier): ~$0-2/mo
  • 20GB R2 storage: ~$0.30/mo
  • Buffer for bandwidth/operations: ~$1-2/mo

    • Why $20/mo for Web Access (not $15)?
  • At $15/mo with $8 cost = 47% margin (too thin)
  • At $20/mo with $8 cost = 60% margin (sustainable)
  • Competitors: Notion $10, Roam $15, Evernote $15 (but no dedicated VPS)
  • We're offering a dedicated instance with local data + agents - worth the premium
    • Alternative: Web Access Pro at $35/mo

  • Larger instance (2GB RAM)
  • For power users who need more resources
    • OpenClaw Users: Self-Hosted Onelist
  • Users running their own OpenClaw instance already have a VPS
  • They can install Onelist on the same machine (self-hosted, $0)
  • Full Onelist Web + agents without Web Access subscription
  • Recommended: Cloud Sync ($3/mo) for backup and multi-device sync
  • This is a key adoption path - OpenClaw users get Onelist "for free" on their existing infra

2. Create B2 bucket: onelist-backup-prod
3. Create application key (S3 compatible API)
4. Note endpoint: s3.{region}.backblazeb2.com
5. Configure bucket lifecycle (if using for archive)

2. Install flyctl CLI

6. Set environment secrets
7. Deploy: fly deploy
