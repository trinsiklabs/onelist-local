River Agent UI Plan

Document Version: 2026-01-30 Status: Active Parent: ui_plan.md Related: shared_agent_ui_components.md, settings_ui_plan.md

1. Overview

This document defines UI components specific to the River Agent. For shared agent UI patterns (processing status, activity items, cost display), see shared_agent_ui_components.md.

River Responsibilities:
  • Natural language queries over user's knowledge base
  • GTD-based task and project management
  • Intelligent entry filing and classification
  • Proactive reminders and coaching
  • Review workflows (Daily, Weekly)
  • Orchestration of other agents
Key Design Principle: River is the ONE assistant users talk to - a unified conversational interface, not multiple agents. GTD Foundation: River implements Getting Things Done methodology:
GTD StepRiver Implementation
CaptureInbox for all incoming entries
ClarifyProcess entries, identify next actions
OrganizeDomain/Scope/Project hierarchy + GTD buckets
ReflectDaily, Weekly, Monthly reviews
EngageContext-based views, focus modes

2. Chat Interface

Primary interaction surface for River. All River interactions (user-initiated and proactive) flow through a unified conversation.
